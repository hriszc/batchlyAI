# Studio Portrait Recraft 项目技术总结

这是一套适合复用到其他 AI 工具站的轻量架构：静态前端 + Cloudflare Worker + Cloudflare KV + 邀请码额度 + 手机端优先体验。

## 核心架构

- 前端是纯静态页面，使用 HTML/CSS/JavaScript，无构建链，部署和迁移成本低。
- 后端使用 Cloudflare Worker，负责代理 AI API、隐藏 API Key、统一解析返回、限流、邀请码和图片代理。
- 静态资源和 Worker 一起部署到 Cloudflare Workers Assets。
- 数据存储使用 Cloudflare KV，保存邀请码、使用次数、心愿单邮箱和每日限流记录。

## 产品机制

- 手机端优先：上传、生成、保存、分享都围绕手机浏览器体验设计。
- 支持批量上传：单次最多 9 张。
- 批量生成采用前端逐张短请求，而不是一个长请求处理多张，减少连接中断和重复生成风险。
- 生成结果逐张展示，单张失败不影响其他图片。
- 进度条用于缓解等待焦虑，按预计耗时持续推进。
- 生成完成后自动滚动到结果区，减少用户找结果的成本。
- 保存图片通过图片代理拉取 Blob 后下载，避免跨域图片在当前页打开导致页面状态丢失。
- 分享图使用 Canvas 合成前后对比图，并添加水印 `spr.pikafun.com`。

## 增长与风控

- 心愿单功能收集用户邮箱，为后续高额度、批量能力、付费转化保留入口。
- 邀请码使用 Cloudflare KV 存储，每个邀请码有 `limit` 和 `used`。
- 邀请码可选：不填邀请码走每日免费额度，填写有效邀请码则跳过每日免费限制，按邀请码额度扣次数。
- 防盗刷基于 `IP + clientId + User-Agent` 生成匿名 hash，限制每位用户每日免费使用次数。
- 批量生成按成功张数扣额度，避免失败也扣次数。

## 稳定性经验

- 不要让 Worker 在一次请求里串行等待多张 AI 生成任务，容易出现连接关闭。
- 批量任务更适合前端拆成多个短请求，逐张执行、逐张更新 UI。
- 第三方 API 返回格式需要做容错解析，兼容 JSON、`raw` 字符串、`data:` 流式文本和 `results[0].url` 等格式。
- 前端不能假设接口一定返回 JSON，遇到 HTML 或非 JSON 时要给清晰错误提示。
- 本地 `file://` 调试需要单独处理 API Origin 和 CORS。
- 跨域图片不能直接依赖 `<a download>`，应通过后端图片代理转 Blob 下载。

## 可复用场景

这套模式适合：

- AI 图片处理工具
- AI 头像、证件照、简历照生成工具
- AI 文案、简历、海报等轻量工具站
- 免费试用 + 邀请码额度制产品
- 手机端传播型小工具
- 无数据库、低成本 MVP

## 一句话总结

这是一个“静态前端 + Cloudflare Worker + KV + 邀请码额度 + 手机端优先 + AI API 代理”的轻量 AI 工具站架构。
